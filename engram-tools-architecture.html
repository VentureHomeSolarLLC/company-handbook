Tools in the Engram: The "Job Box" Philosophy

1. The Brain vs. The Hands

In the Agora architecture, Engrams are the "Brain" (Knowledge, Logic, Rules) and Tools are the "Hands" (Capabilities, Actions).

An Engram doesn't *do* anything itself. It is a set of instructions that tells the Agent how to use its Tools.

The Contractor Analogy

Imagine an Agent is a Contractor showing up to a job site.

  • The Agent: The person.
  • The Tools: The hammer, drill, and wrench on their belt.
  • The Engram: The blueprint and the instruction manual for the specific house they are building.

The Blueprint (Engram) tells the Contractor (Agent) to "Use the Drill (Tool) on the Wall."

2. Global Tools vs. Local Tools

Not all tools are created equal. Some are universal; some are hyper-specialized. We manage this distinction to keep agents lightweight but capable.

Type A: Global Tools (The Tool Belt)

These are the tools every agent has by default. They are part of the OpenClaw runtime, not the Engram.

  • Examples: read_file, write_file, web_search, browser_click.
  • Location: System Level (Runtime).
  • Usage: Any Engram can call these at any time.

Type B: Local Tools (The Job Box)

Sometimes, a job requires a specialized instrument that isn't standard issue. We don't want to weigh down every agent with a "Voltage Drop Calculator" or a "CAD Rendering Script." We only want those tools present when the specific job demands them.

These are Local Tools, and they live inside the Engram.

  • Examples: calculate_string_sizing.py, generate_line_diagram.js, parse_enphase_csv.py.
  • Location: Inside the Engram directory (`engrams/engineering/tools/`).
  • Usage: Only available when the Agent loads this specific Engram.

3. The "Job Box" Architecture

Housing specialized tools inside the Engram makes the Engram portable and self-contained. Everything needed to complete the task—the instructions, the data, and the specialized equipment—is in one folder.

engrams/solar-engineering/ ├── _index.md <-- The Router ├── skills/ │ └── calculate-string-sizing.md <-- The Procedure (Step 1: Run Calculator) ├── concepts/ │ └── nec-codes.md <-- The Reference Data (Rules) └── tools/ <-- THE JOB BOX └── voltage-drop-calculator.py <-- The Specialized Tool

4. How an Agent Uses a Local Tool

When an agent executes the Skill, it sees an instruction to use the local tool.

# SKILL: Calculate String Sizing 1. **Gather Data:** Read the panel specifications from `concepts/panels.md`. 2. **Execute Tool:** Run the local script `tools/voltage-drop-calculator.py` with the panel data as input. - Command: `exec(command="python3 engrams/solar-engineering/tools/voltage-drop-calculator.py --volts 400")` 3. **Interpret Output:** Read the JSON output from the script. 4. **Pivot:** Compare output against `concepts/nec-codes.md`.
