Addendum to Project Agora: The Evolution of Forge

The Project Agora brief describes the agent's world. This document details how **Forge must evolve alongside the agents** to support that vision. Forge cannot remain a simple task tracker. It must become the central nervous system for our entire AI-human workforceâ€”the operating system for the Agora itself.

1. Structured Tasking: From Text Box to Command

The root of the "synaptic gap" is that a task in Forge is unstructured text. To solve this, we must transform tasks into structured commands that our agents can understand natively.

What Needs to Change:

  • New Field: "Primary Engram": A dropdown list, automatically populated from our engrams/ directories, allows a user to link a task directly to the knowledge required to complete it.
  • New Feature: "Task Templates": For common tasks, a template can pre-select the Engram and required parameters, removing cognitive load.
  • New Field: "Structured Parameters": Dedicated, machine-readable fields for crucial data like project_id: "12345".
  • Granular Statuses: Evolve beyond "To Do / Done" to include states like Pending Data Verification, Awaiting Customer Signature, or Blocked: Missing Information.

The Result: An agent's heartbeat no longer finds a vague instruction. It finds a structured JSON object: { task: "Submit Interconnection", engram: "interconnection-submission", params: { project_id: "12345" } }. There is no room for misinterpretation. The path to execution is crystal clear.

2. Agent Management: Visualizing the Hierarchy

Our workforce will consist of Manager agents and their permanent Worker Bee sub-agents. The Forge UI needs to reflect this reality.

What Needs to Change:

  • Org Chart View: The "AI Operations" page will include an organizational chart view, showing which Worker Bees report to which Manager.
  • Specialization Tag: Each Worker Bee profile will clearly display its specialization, directly tied to the primary Engram it was designed to execute (e.g., "Specialization: interconnection-submission").

The Result: We gain a clear, visual understanding of our AI workforce's structure, capabilities, and reporting lines, making it easy to see who is responsible for what.

3. The Agora Interface: A Library for Our Collective Brain

The Engrams cannot live only in GitHub. They must be visible, navigable, and understandable for the entire team directly within Forge.

What Needs to Change:

  • New Top-Level Section: "The Agora": A new tab in the Forge sidebar.
  • Engram Browser: An interface that reads directly from our engrams/ Git repositories and displays them in a clean, navigable UI.
  • Mermaid.js Rendering: When a user clicks on an Engram's MOC (_index.md), Forge will parse and render the embedded Mermaid.js diagrams, providing an instant visual understanding of the process flow.
  • Integrated "Lessons" View: Each Engram page will have a dedicated tab to view all the "Lessons" associated with it, allowing any team member to review the history of real-world learnings.

The Result: The Agora becomes the living, breathing heart of our operations. It's not just for agents; it's the central knowledge base and training manual for the entire company.

4. The Feedback Loop: Closing the Circuit of Learning

We must formalize the process of turning human feedback into machine-readable lessons.

What Needs to Change:

  • New Button: "Provide Feedback": On every completed task, a new button appears. Clicking it opens a simple form: "What was the unexpected outcome? What was the resolution?"
  • New Task State: `Pending Lesson Review`:** When a human submits feedback, the original task's status changes. This becomes the **trigger** for the Scribe agent.
  • New UI: "Pending Lessons" Queue:** Within the Agora interface, a new queue appears, showing all the lessons that have been auto-generated by Scribe agents and are awaiting review.
  • Integrated PR Review: Each item in the queue links directly to the pull request in GitHub that the Scribe created. A VH1 user can review, approve, and merge the lesson directly from the Forge UI.

The Result: We create a seamless, end-to-end workflow for institutional learning. The friction of capturing knowledge is removed. The process of improving our automation becomes, itself, automated.
